# ...进程... #
Process，是操作系统中的基本执行单位，是程序在计算机上的一次动态执行程。每个进程都有独立的内存空间、文件描述符、环境变量等资源，操作系统通过进程管理来实现多任务并发执行。是资源分配的基本单位：操作系统为每个进程分配独立的资源（如内存、CPU 时间）。
# ...线程... #
线程（Thread）是操作系统能够进行运算调度的最小单位，它被包含在进程之中，是进程中的实际运作单位。一个进程可以包含多个线程，这些线程共享进程的资源，但每个线程有自己的执行上下文（如程序计数器、寄存器、栈）。线程可以在同一进程内并发执行，提高程序的响应速度和处理能力
线程的缺点：
同步问题：线程共享资源可能导致竞争条件，需要使用同步机制（如互斥锁、信号量）来避免。
调试困难：多线程程序的调试和测试比单线程程序更复杂。
容错性低：一个线程的崩溃可能导致整个进程崩溃
# Daemon #
守护进程（Daemon）是在后台运行的一种特殊进程，通常用于执行系统任务或服务，而不需要用户交互。它们通常在系统启动时启动，并在系统运行期间持续运行

# ...内核态...&...用户态... #
内核态：内核空间，内核进程和线程所在区域
用户态：用户空间，用户进程和线程所在区域
区分两个态是为了保护设备，防止进程错误操作或恶意破坏。
用户态切换到内核态的方法：
1....系统调用（System Call）
系统调用是用户态程序主动请求操作系统服务的方式。通过系统调用，程序可以访问文件、创建进程、进行网络通信等。
工作原理
用户态程序执行特定的指令（如int9x89或syscall），触发软中断。
2．CPU切换到内核态，并啭到内核中预定义的系统调用处理函数。
3．内核执行请求的操作，并将结果返回给用户态程序。
4．CPU切换回用户态，程序继续执行。

2. 中断（Interrupt）
中断是由硬件或软件触发的事件，用于通知 CPU 需要立即处理某些任务。当中断发生时，CPU 会暂停当前任务，切换到内核态处理中断。
硬件中断：由硬件设备触发，如键盘输入、网络数据到达。
软件中断：由程序触发，如系统调用。
类型
硬件设备或软件触发中断。
CPU 保存当前上下文（如寄存器、程序计数器）。
CPU 切换到内核态，并跳转到中断处理程序（Interrupt Service Routine, ISR）。
内核处理中断，完成后恢复上下文并返回用户态。
工作原理
键盘输入：用户按下键盘时，触发硬件中断，内核处理输入事件。
定时器中断：定时器到期时，触发中断，内核进行任务调度。
示例
3. 异常（Exception）
异常是由程序执行过程中发生的错误或特殊情况触发的，如除零错误、页面错误（Page Fault）。
程序执行过程中发生异常。
CPU 保存当前上下文，并切换到内核态。
内核处理异常，可能终止程序或修复错误。
如果异常被修复，CPU 恢复上下文并返回用户态。
工作原理
页面错误：访问未映射的内存时触发，内核可能加载缺失的页面。
除零错误：除数为零时触发，内核通常终止程序。
常见异常
4. 陷阱（Trap）
陷阱是一种特殊的软中断，通常用于调试或系统调用。与中断不同，陷阱是同步的，由程序主动触发。
程序执行陷阱指令（如 int 3）。
CPU 切换到内核态，并跳转到陷阱处理程序。
内核处理陷阱，完成后返回用户态。
工作原理
调试：调试器使用陷阱指令设置断点。
系统调用：某些架构使用陷阱指令实现系统调用。
使用场景
5. 上下文切换（Context Switch）
上下文切换是多任务操作系统的核心机制，允许 CPU 在多个进程之间切换。当发生上下文切换时，CPU 会从用户态切换到内核态。
当前进程的时间片用完或主动让出 CPU。
CPU 保存当前进程的上下文，并切换到内核态。
内核选择下一个要运行的进程，并加载其上下文。
CPU 切换回用户态，运行新进程。
工作原理

# ...RAID...  #
Redundant Array of Independent Disks，独立磁盘冗余阵列.... ...是一种将多个物理磁盘组合成一个逻辑单元的技术，旨在提高数据存储的性能、可靠性和容量。
工作原理：
条带化（Striping）：将数据分块并分布到多个磁盘上，提高性能。
镜像（Mirroring）：将数据复制到多个磁盘上，提高可靠性。
奇偶校验（Parity）：通过计算校验信息实现数据冗余，允许磁盘故障后恢复数据
级别：
RAID0
·特点
·优点
·缺点
（条带化）
将数据分块并交替存储到多个磁盘上。
提高读写性能。
无冗余，单个磁盘故障会导致数据丢失。
·适用场景：需要高性能但不要求数据冗余的场景（如临时数据存储）。
RAID1（镜像）
·特点将数据完全复制到两个或多个磁盘上。
·优点高可靠性，单个磁盘故障不影响数据。
·缺点存储利用率低（50％）。
·适用场景．需要高可靠性的场景（如关数据存储）。
RAID5
·特点
·优点
·缺点
（带奇偶校验的条带化）
将数据和奇偶校验信息分布到多个磁盘上。
兼顾性能、容量和可靠性。
写入性能较低，重建时间较长。
·适用场景．需要平衡性能、容量和可靠性的场景（如文件服务器）。
RAID6
·特点
·优点
·缺点
（双奇偶校验）
使用两个犭虫立的奇偶校验块，允许两个磁盘同时故障。
更高的可靠性。
写入性能更低，存储利用率较低。
·适用场景．需要更高可靠性的场景（如大型存储系统）。
RAID10（镜像+条带化）
·特点
结合RAID1和RAID0，先镜像再条带化。
·优点
高性能和高可靠性。
·缺点
成本高，存储利用率低（50％）。
适用场景．需要高性能和高可靠性的场景（如数据库服务器）。
# ...软链接...&...硬链接...#
软链接是一个特殊的文件，它指向另一个文件或目录的路径。类似于 Windows 中的“快捷方式”。
特点
·存储内容．存储的是目标文件的路径（而不是文件内容）。
·跨文件系统．可以链接到不同文亻牛系统的文亻牛或目录。
·链接目标．可以链接到文亻牛或目录。
·删除目标文件：如果目标文件被删除，软链接会变成"悬空链接"（失效）。
·权限．软链接的权限是777（所有用户可读、写、执行），但实际权限由目标文件决定。

硬链接是文件系统中对同一个文件的多个引用。所有硬链接指向同一个 inode（文件系统中的索引节点）。
特点
存储内容．直接指向目标文件的inode,而不是路径。
·同一文件系统．硬链接之须与目标文亻牛位于同一文亻牛系统。
·链接目标：只能链接到文件，不能链接到目录。
·删除目标文件：即使删除目标文件，只要还有硬链接存在，文件内容仍然保留。

·3、软链接和硬链接的区别：
·1．定义不同
·软链接又叫符号链接，这个文件包含了另一个文件的路径名。可以是任意文件或目
录，可以链接不同文件系统的文件。
．硬链接就是一个文僻的一个或多个文件名·把文件名和计算机文件系统使用的节点
号鲢接起来。因此我们可以用多个文件名与同一个文件进行链接，这些文件名可以
在同一目录或不同目录。
·2．限制不同
．硬链接只能对已存在的文件进行创建，不能交叉文件系统进行硬链接的创建
．软链接可对不存在的文件或目录创建软链接，可交叉文件系统，
3．创建的方式不同
硬链接不能对目录进行创建，只可对文件创建
软链接可对文件或目录创建
·4．影响不同
．删除一个硬链接文件并不影响其他有相同inode号的文件·
．删除软链接并不影响被指向的文件，但若被指向的原文件被删除，则相关软连接被
称为死链接
