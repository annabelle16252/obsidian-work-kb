# SA #
协议的核心机制，用于在两个网络设备（如路由器、防火墙、VPN 网关）之间建立安全通信的 加密通道。它定义了如何对数据进行加密、认证和保护，确保传输的机密性、完整性和真实性。
（1）AH（Authentication Header）SA...   已经淘汰
功能：提供 数据完整性 和 来源认证，但不加密数据。
协议号：51
典型用途：适用于无需加密但需防篡改的场景（如内部网络监控）。
（2）ESP（Encapsulating Security Payload）SA
功能：提供 加密（机密性） + 完整性校验 + 认证。
协议号：50
典型用途：VPN 隧道（如 Site-to-Site VPN、远程访问 VPN）。
封装模式：
将安全协议的相关字段插入原始的...IP...报文，实现对报文的加密和认证。
传输模式：将...AH...和...ESP...插入原始报文，不改变...ip...头，适用于安全传输的起点和终点为数据包的实际起点和终点
隧道模式：将...AH...和...ESP...插入报文，并添加新的...IP...头，适用于安全传输的起点和终点为安全网关
